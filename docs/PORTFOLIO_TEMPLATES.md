# Portfolio templates — delivery scope

Career Systems uses **three portfolio shells**. Clients get tailored **content**, not a bespoke design every time.

## Templates

| Slug | Name | Best for | Foundation |
|------|------|----------|:----------:|
| `minimal` | Minimal | Simple proof-of-work sites | ✓ |
| `data-tech` | Data-Tech | Engineering / data roles | ✓ |
| `professional` | Professional | Business / ops positioning | Launch+ only |

## What is included (all packages)

- Pick template at Analysis (client preference from intake or operator override)
- Fill template slots: name, target role, projects, skills, links
- Deploy to GitHub Pages or Render subdomain
- Light tweaks: accent color, section order within template, copy emphasis

## What is **not** included (quote separately or decline)

- New page types not in the template
- Full redesign after Build approval
- “Copy this designer’s portfolio” requests
- Unlimited custom layout per client

## Package → template rule

- **Foundation:** `minimal` or `data-tech` only
- **Launch / Accelerator:** any template + stronger narrative alignment

## Operator workflow

1. Intake captures optional **Portfolio template preference**
2. On intake complete, project auto-moves to **Analysis**
3. Complete task **Select portfolio template** (use recommendation in client detail when wired)
4. Build from `templates/portfolio/` starter (scaffold when ready)

Registry: `app/template_config.py`
